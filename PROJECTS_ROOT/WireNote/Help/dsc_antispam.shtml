Anti-SPAM Module

This feature will help you to free your POP3 mailboxes from SPAM (aka Junk) emails. WireNote detects SPAM by presence of certain strings in different message fields. Such strings are keeped in spam-string files (one for email subject, one for 'From' field and one for first lines of message body). To enable Anti-SPAM you must define at least one POP3 account. This can be done in the preferences dialog under "Main features"/"Anti-SPAM accounts" folder. By default you have one POP3 account, but you can use anti-SPAM with more accounts if you want to do so - just enter number in corresponding option and restart WireNote. To check POP3 account, first set up server address, port (110 is the standard value), account login, password and check period (1 hour by default). By default Spam checker works in training mode. In this mode WireNote will open special window with received message headers every time it found new email in your mailbox. You will be able to manually check SPAM messages and add corresponding text to spam-string files. Every line in such file define word or string which is undoubtedly classify message as SPAM. If you don`t want to do this manually, you can find such files in the Internet for popular email client TheBAT! In other modes WireNote will automatically kill SPAM messages on server and optionally start default email client after such clean up (if your mailbox will contain non-SPAM messages)

Tips and tricks
  • Double click message header to view message`s RFC 822 headers.
  • You can find why message was detected as SPAM by holding mouse pointer on its header - WireNote will show popup with detailed message description.
  • When you are clicking on message field, its content will be automatically put in the "Add SPAM-strings" edit field
  • You can run check manually - just open WireNote tray menu and go to the "Utilities". You will find here one item per properly configured POP3 account. Select account you want to check and press "Check now" button
